Mockup generation spec

Rules for any product mockup generated for Noon (Claude Design or elsewhere). This is the distilled GLOBAL BLOCK that produced Harborline, Quillshore, Gantrey, Ledgermark and the six /work cases.

Deliverable
  • ONE self-contained HTML file. Inline CSS/JS/SVG; the only external reference allowed is the Google Fonts <link> (declare every family + weight). Icons as inline SVG; avatars as initials circles; charts drawn in CSS/SVG. No external images, no icon CDNs.
  • Authoring viewport exactly 1440px wide, natural height 900–1050px. Desktop only — no phone/tablet variants.
  • Edge to edge: body{margin:0}, NO browser chrome, NO window frame, NO drop shadow, NO rounded outer corners, NO backdrop. The app's own background is the outermost surface.
Quality bar
  • Linear/Vercel-grade density: a real product screenshot, not a marketing illustration — status chips, hairline table rules, mono micro-labels, toolbars, filters, pagination, timestamps, one active nav item.
  • Believable data everywhere — never “Lorem”, never “Item 1/2/3”.
  • One identity per product: one accent + semantic green/amber/red, one distinctive Google Font pairing (UI + mono). Must not look like a Tailwind default.
  • Live :hover states; tiny vanilla JS for hover/tab states only; no looping/autoplay animations.
Legibility floor (it displays scaled to ~50–60%)
  • Body/table text ≥ 12.5px at authoring size · micro-labels ≥ 10px · WCAG-AA-ish contrast · no ultra-light weights under 13px.
Content rules
  • ALL UI copy in English. Fictional product + people names only; verify product names don't collide with real software companies (Ordwell, Crewfield, Lotvane, Stackbrief, Visitra, Embertide, Harborline, Quillshore, Gantrey, Ledgermark are taken & verified).
  • No real brands/logos inside the UI (generic phrases like “ERP sync” are fine). Realistic-but-fictional metrics are fine INSIDE the fictional UI.
  • Top-of-file manifest comment: product name, authoring viewport (1440×H), fonts, accent hex.
Handoff
  • If the tool produces a .dc.html + runtime, always request: “bundle it into ONE standalone, fully self-contained HTML file”. Delivery pipeline self-hosts the fonts and embeds it as a constant-viewport iframe.
