Brand rules — the non-negotiables

Always
  • The wedge is the protagonist: every build human-reviewed — a PM approves proposals, a senior engineer signs code. Show it as gates/receipts, don't just say it.
  • Honest by construction: only real, verifiable, attributed claims. Illustrative artifacts carry a visible “Illustrative” tag; case interfaces are anonymized recreations and say so.
  • Flat + hairline + one accent (#1200c5) with the semantic pair (success #2cc49a / warning #f0a127).
  • Motion encodes a meaning, then stops. Reveal once on scroll; reduced-motion sees everything instantly (static final frame).
  • Sentence case everywhere; paired-clause headlines where they earn it (“AI speed. Human judgment.”).
  • “Built on” authority: Vercel/Stripe/Supabase as capability facts.
  • Both themes always: light #fff/#000 · dark #000/#e8edf7 — tokens, never hardcoded.
Never
  • No invented numbers — metrics, testimonials, logos, certifications, or clock promises (“in minutes” is banned as toy phrasing; turnaround stays qualitative).
  • No “trusted by / partners with” — the ecosystem doesn't endorse Noon until contractually true.
  • No decorative motion: no infinite loops, floating elements, glows, gradients, mesh backgrounds, or futuristic clichés.
  • No model names in the process narrative (the Maxwell explainer says “AI”); stack names live only in case-study stacks and capability strips.
  • No Title Case, no ALL-CAPS marketing, no superlatives (“world-class”, “record time”).
  • No boxes-with-shadows structure — hairlines do the work.
The test for any new piece: would a skeptical CTO believe this on first read — and could we defend every word? If a claim needs a source, link the source (cases link /work, quotes link the original).
